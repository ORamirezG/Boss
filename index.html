Git

Git es un sistema de control de versiones para el mantenimiento eficiente y confiable de archivos.

Git trabaja de forma descentralizada, lo que significa que cada desarrollador tiene una copia completa del historial de versiones de un proyecto, y puede trabajar sin conexión con ese historial.

Nos permite trabajar de forma colaborativa, ya que cada desarrollador puede trabajar en su propia rama y luego fusionar su trabajo con la rama principal.

Instalar Git

Instalar Git

Para instalar Git en Windows, macOS o Linux, debemos descargar el instalador desde la página oficial de Git, https://git-scm.com/downloads .

Una vez descargado el instalador, lo ejecutamos y seguimos los pasos del asistente de instalación.

En Linux podemos instalar git con la terminal de comandos, ejecutando el comando sudo apt-get install git.

Para comprobar que Git se ha instalado correctamente, abrimos una terminal y ejecutamos el comando git --version.

Si se ha instalado correctamente, nos mostrará la versión de Git que tenemos instalada.

Si no se ha instalado correctamente, nos mostrará un mensaje de error.

Debemos configurar Git con nuestro nombre de usuario, correo electrónico y nuestro editor de texto por defecto, para que aparezcan en los commits que hagamos. Para ello debemos estar registrados en GitHub, GitLab o Bitbucket.

El comando git config --global user.name "nombreUsuario" nos permite especificar nuestro nombre de usuario. Esta información aparecerá en los commits.

El comando git config --global user.email "correoElectronico" nos permite especificar nuestro correo electrónico. Esta información aparecerá en los commits.

El comando git config --global init.defaultBranch main nos permite especificar el nombre de la rama principal. Por convenio se usa main en lugar de master.

El comando git config --global core.editor "code --wait" nos permite especificar nuestro editor de texto por defecto. En este caso, code indica que nuestro editor de texto por defecto es Visual Studio Code y --wait indica que Git debe esperar a que cerremos el archivo que estamos editando para continuar. Para los que usais Mac sería con el comando git config --global core.editor "open -a 'Visual Studio Code' --wait".

El comando git config --global -e nos permite editar el archivo de configuración de Git en VS Code.

El comando git config --global core.autocrlf valor nos permite especificar si queremos que Git convierta los saltos de línea de Windows a saltos de línea de Linux/Mac o viceversa. El valor debe ser true para los que esten trabajando en Windows e input para los que esten trabajando en Linux/Mac.

Para poder investigar mas a fondo la configuración de git podemos usar el comando git config -h, esto nos mostrara una lista de comandos que podemos usar para configurar git.

git config --global user.name "nombreUsuario" git config --global user.email "correoElectronico" git config --global init.defaultBranch main git config --global core.editor "code --wait" git config --global core.editor "open -a 'Visual Studio Code' --wait" git config --global -e git config --global core.autocrlf valor
SSH

SSH

SSH es un protocolo de red que nos permite conectarnos a un servidor remoto de forma segura.

Para conectarnos a un servidor remoto con SSH, debemos generar una clave pública y privada.

Para generar una clave pública y privada, abrimos una terminal y ejecutamos el comando ssh-keygen -t ed25519 -C "tu-email@dominio.com".

El comando ssh-keygen nos permite generar una clave pública y privada.

El parámetro -t ed25519 nos permite especificar el tipo de clave que queremos generar. En este caso, ed25519 es el tipo de clave que queremos generar.

El parámetro -C "correoElectronico" nos permite especificar un comentario que queramos añadir a la clave. En este caso, "correoElectronico" es el comentario que queremos añadir a la clave.

El comando nos preguntará dónde queremos guardar la clave. Si pulsamos la tecla Enter, se guardará en la ruta por defecto.

El comando nos preguntará si queremos añadir una contraseña a la clave. Si pulsamos la tecla Enter, no añadiremos contraseña a la clave.

Una vez generado el par de claves, debemos añadir la clave pública a nuestro servidor remoto.

Para añadir la clave pública a nuestro servidor remoto, debemos copiar el contenido de la clave pública.

Para copiar el contenido de la clave pública, abrimos una terminal y ejecutamos el comando cat ~/.ssh/id_ed25519.pub.

El comando cat nos permite mostrar el contenido de un archivo.

El parámetro ~/.ssh/id_ed25519.pub nos permite especificar la ruta del archivo que queremos mostrar. En este caso, ~/.ssh/id_ed25519.pub es la ruta del archivo que queremos mostrar.

El comando nos mostrará el contenido de la clave pública.

Una vez copiado el contenido de la clave pública, debemos añadirlo a nuestro servidor remoto.

Para añadir la clave pública a nuestro servidor remoto, debemos ir a la interfaz web de nuestro servidor remoto y añadir la clave pública.

Una vez añadida la clave pública a nuestro servidor remoto, ya podemos conectarnos a nuestro servidor remoto con SSH.

Conceptos básicos de git

Conceptos básicos

  • Repositorio

    Al trabajar con git lo que hacemos es crear un Repositorio.

    Un Repositorio es el lugar donde se almacenan los archivos de un proyecto. Puede ser local o remoto.

    • Repositorio local

      Es el creado de forma local en nuestro ordenador.

      Se crea en la carpeta del proyecto con el comando git init.

    • Repositorio remoto

      Se crea en un servidor externo como GitHub, GitLab o Bitbucket.

  • Etapas

    Es el proceso que sigue git para llevar a cabo las acciones de add, commit y push.

    • Working directory

      Es el directorio de trabajo. Es el lugar donde se encuentran los archivos del proyecto.

    • Staging area

      Es el área de preparación. Es el lugar donde se encuentran los archivos que se van a añadir al repositorio.

    • Commit

      Es el lugar donde se encuentran los archivos del proyecto que están listos para ser enviados al repositorio remoto.

      La Statging area estaría vacía.

    • Server

      Es el Repositorio remoto donde se encuentran los archivos del proyecto.

  • Add

    Es el proceso de añadir los archivos al área de preparación. Se puede comparar con el proceso de añadir los archivos a una cesta de la compra.

  • Commit

    Es una instantánea de los archivos del proyecto en un momento determinado. Se puede comparar con una fotografía de los archivos en ese momento.

  • Branch

    Es una rama o línea de desarrollo independiente. Cada rama contiene una serie de commits.

  • Merge

    Es la fusión o unión de dos ramas. Se puede comparar con la fusión de dos ramas de un árbol genealógico.

  • Checkout

    Es el proceso de moverse entre ramas. Se puede comparar con cambiar de página en un libro.

  • Push

    Es el proceso de subir los commits locales a un repositorio remoto.

  • Pull Request

    Es la solicitud de incorporar los commits de una rama a otra.

  • Pull

    Es el proceso de traer los commits de un repositorio remoto a un repositorio local.

  • Clone

    Es el proceso de copiar un repositorio remoto a un repositorio local.

  • Fetch

    Es el proceso de traer los commits de un repositorio remoto a un repositorio local sin fusionarlos con la rama actual.

  • Stash

    Es el proceso de guardar los cambios de la rama actual en una pila de cambios para poder trabajar en otra rama.

Crear un repositorio

Crear un repositorio

  • Plataforma

    Se puede crear un repositorio en las siguientes plataformas: GitHub, GitLab o Bitbucket.

  • Repositorio remoto

    1. Para crear un repositorio remoto debemos acceder a la interfaz web de GitHub, GitLab o Bitbucket y crear un nuevo repositorio.

    2. Una vez creado el repositorio, debemos clonar el repositorio remoto en un repositorio local. Esto lo podemos hacer por HTTPS o por SSH.

      • Clonar repositorio remoto por HTTPS
        git clone https://github.com/LuisHackABoss/pruebasGit.git
      • Clonar repositorio remoto por SSH
        git clone git@github.com:LuisHackABoss/pruebasGit.git

  • Repositorio local

    1. Para crear un repositorio local debemos situarnos en la carpeta del proyecto y ejecutar el comando git init.

      Si no hay carpeta del proyecto podemos crearla con el comando mkdir nombreCarpeta, situarnos en ella con el comando cd nombreCarpeta y ejecutar el comando git init.

      No influye si la carpeta está vacía o contiene archivos.

      Este comando crea una carpeta oculta llamada .git que contiene toda la información del repositorio.

      Para comprobar que se ha creado el repositorio local, podemos ejecutar el comando ls -a para listar los archivos y carpetas ocultas y comprobar que existe la carpeta .git.

    2. Tenemos que crear un repositorio remoto en GitHub, GitLab o Bitbucket para poder vincularle el repositorio local.

      Una vez creado el repositorio remoto, lo vinculamos con el repositorio local por primera vez con el comando git remote add origin git@github.com:LuisHackABoss/nombreRepositorio si lo hacemos por HTTPS o git remote add origin git@github.com:nombreUsuario/nombreRepositori en el GitHub si lo hacemos por SSH.

    3. Para subir los commits locales al repositorio remoto, debemos ejecutar el comando git push -u origin main.

Git Add

Git Add

Añadimos los cambios de un archivo a la etapa de staged con el comando git add nombreArchivo.

Si quiero añadir varios archivos a la etapa de staged, debemos ejecutar el comando git add nombreArchivo1 nombreArchivo2.

Si queremos añadir todos los cambios de todos los archivos de una extensión a la etapa de staged, debemos ejecutar el comando git add *.extensión.

Si queremos añadir todos los cambios de todos los archivos a la etapa de staged, debemos ejecutar el comando git add ..

Si ejecutamos el comando git status nos mostrará que no hay commits todavía y que el archivo README.md está preparado para el commit.

Si ejecutamos el comando git log nos mostrará que no hay commits todavía.

Git Commit

Git Commit

Para crear el primer commit, debemos ejecutar el comando git commit -m "first commit".

El -m indica que lo siguiente es el mensaje del commit.

Si ejecutamos el comando git status nos mostrará que en la rama main no hay nada para hacer commit, la Staging area está vacía y el árbol de trabajo está limpio.

Hay commits en el repositorio local, pero no en el repositorio remoto.

Si ejecutamos el comando git log nos mostrará el hash del commit, el autor, la fecha y el mensaje del commit.

Cambios, add y commit

Cambios, add y commit

Si hacemos cambios en el archivo README.md y queremos añadirlos al repositorio local, debemos ejecutar los comandos git add . y git commit -m "mensaje".

Si ejecutamos el comando git log o git log --oneline nos mostrará los commits realizados que se han ido añadiendo al repositorio local.

Si me he confundido y quiero cambiar el mensaje del último commit, debemos ejecutar el comando git commit --amend -m "mensaje".

Si quiero deshacer el último commit pero mantener los cambios en el área de trabajo, debemos ejecutar el comando git reset --soft HEAD^.

Si quiero deshacer el último commit sin mantener los cambios en el área de trabajo, debemos ejecutar el comando git reset --hard HEAD^.

Ignorar archivos y directorios

Ignorar archivos y directorios

Para ignorar archivos y directorios debemos crear un archivo llamado .gitignore en la raíz del proyecto.

En el archivo .gitignore debemos escribir los nombres de los archivos y directorios que queremos ignorar.

Si queremos ignorar todos los archivos de un tipo, debemos escribir el nombre del tipo de archivo precedido de un punto como por ejemplo .env.

Si queremos ignorar todos los archivos de un directorio, debemos escribir el nombre del directorio precedido de una barra como por ejemplo node_modules.

Si queremos ignorar todos los archivos de un tipo que se encuentren en un directorio, debemos escribir el nombre del directorio seguido de una barra y el nombre del tipo de archivo precedido de un punto como por ejemplo img/.png.

.env node_modules img/.png
Status simplificado

Status simplificado

Para ver el status simplificado de nuestro repositorio local debemos ejecutar el comando git status -s.

Si queremos ver el status simplificado de un archivo en concreto, debemos ejecutar el comando git status -s nombreArchivo.

Si queremos ver el status simplificado de un directorio en concreto, debemos ejecutar el comando git status -s nombreDirectorio.

Historial

Historial

Para ver el historial de nuestro repositorio local, debemos ejecutar el comando git log.

Si queremos ver el historial de un archivo en concreto, debemos ejecutar el comando git log nombreArchivo.

Si queremos ver el historial de un directorio en concreto, debemos ejecutar el comando git log nombreDirectorio.

Si queremos ver el historial simplificado de nuestro repositorio local, debemos ejecutar el comando git log --oneline.

Ver cambios

Ver cambios

Para ver los cambios que hemos realizado en un archivo, debemos ejecutar el comando git diff nombreArchivo.

Si queremos ver los cambios que hemos realizado en un directorio, debemos ejecutar el comando git diff nombreDirectorio.

Si queremos ver los cambios que hemos realizado en un commit, debemos conseguir primero la id del commit con el comando git log y despues ejecutar el comando git diff idCommit.

Si queremos ver los cambios que hemos realizado en la etapa staging, debemos ejecutar el comando git diff --staged.
